Getting Started

p0ckit is a Bash-based framework designed to manage security modules. It allows you to standardize your toolkit and run complex scripts through a unified interface.

⚠️ WARNING: For educational and authorized testing purposes only.

Installation

Linux/Unix

$ git clone https://github.com/p0ckit/p0ckit.git
$ cd p0ckit
$ bash p0ckit.sh

Docker (Windows/macOS)

If you are on a non-Linux system, use the provided Docker container:

$ docker build -t p0ckit .
$ docker run -i -t p0ckit

Basic Usage

Launch the framework by running p0ckit.sh. You will be greeted by the fw() prompt.

1. Loading a Module

fw()# use core/ntscan
fw(core/ntscan)#

2. Configuring Options

fw(core/ntscan)# set rhost 192.168.1.1
fw(core/ntscan)# set port 80

3. Execution

fw(core/ntscan)# run

Module Reference

Detailed descriptions of all built-in modules can be found in the [modules] section.

Custom Modules

You can add your own modules to the framework. Each module must follow the p0ckit metadata format for options and info.

#str_op
#   option1
#   option2
#end_op

#str_info
#   Module description here
#end_info

Execution Model

Each module must define its execution timing using one of the following:

#t first  # Executes before variables are set
#t last   # Executes after variables are set
